batch job

  • 把要跑的東西，寫成一個 main.R
  • 這個 main.R 裡面，要多 print 出一些東西
    • 執行一個區塊前，print出正在執行啥區塊。e.g. print出接下來要做的是寫進資料庫
    • 在裡面加上 tryCatch的功能，error出現時，就寄送 automail 到信箱，就知道這個 main.R 沒有 100% 成功執行
    • 如果 main.R 裡面做很多事，不想一出錯就寄 automail，那可以在做各件事時，先用 tryCatch，然後 error 時只是記錄錯在哪(蒐集錯誤訊息或自己定義錯誤訊息)，等到 main.R 的做後一行，再寫條件判斷，如果剛剛有出現過錯誤訊息，那我就寄送 automail給自己
  • 把 main.R 包到 linux 的 .sh 檔中，在 .sh 檔中
    • 叫他去執行 main.R
    • 把執行過程中，console出現的訊息，都存出來到一個檔案(存 log)，例如 output_log
    • 把這個 log，改檔名(e.g. 20210810_output_log)日期，挪到另一個專門放 log 的資料夾
  • 用 crontab訂排程去跑這個 .sh 檔，並把執行的 log 存出來 (這個 log，就是跑這段 main.R 的過程中，在 console 裡出現的訊息)
